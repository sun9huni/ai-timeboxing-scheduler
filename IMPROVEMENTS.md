# 🚀 AI 타임박싱 스케줄러 개선 방안 5가지

## 1. 🔐 API 키 세션 저장 및 환경 변수 지원
**문제점:**
- OpenAI API 키를 매번 입력해야 함
- 보안상 환경 변수 지원이 없음

**개선 방안:**
- API 키를 `st.session_state`에 저장하여 재입력 불필요
- `.streamlit/secrets.toml` 파일로 API 키 관리
- 환경 변수(`OPENAI_API_KEY`) 지원 추가
- 암호화된 로컬 저장소 옵션 제공

**예상 효과:**
- 사용 편의성 향상
- 보안 강화
- 프로덕션 환경 배포 용이

---

## 2. 📊 타임라인 시각화 및 대화형 스케줄 편집
**문제점:**
- 생성된 스케줄을 시각적으로 확인하기 어려움
- 만족스럽지 않은 스케줄 수정 불가
- 시간대별 일정 분포 파악 어려움

**개선 방안:**
- Gantt 차트 또는 타임라인 차트로 일정 시각화 (Altair/Plotly 활용)
- 드래그 앤 드롭으로 일정 시간 조정 가능
- 개별 일정 삭제/추가 기능
- 시간대별 작업 밀도 히트맵 표시

**예상 효과:**
- 직관적인 스케줄 확인
- 사용자 맞춤형 조정 가능
- 시각적 피드백으로 만족도 향상

---

## 3. 💾 사용자 프로필 저장 및 템플릿 기능
**문제점:**
- 매번 프로필 정보를 입력해야 함
- 여러 역할(직장/프리랜서 등) 전환 불가
- 프로필 히스토리 관리 불가

**개선 방안:**
- 프로필을 로컬 JSON 파일에 저장 (`profiles.json`)
- 여러 프로필 생성 및 관리 (예: "평일 업무", "주말 프로젝트")
- 프로필 템플릿 제공 (PM, 개발자, 디자이너 등)
- 프로필별 통계 및 스케줄 히스토리

**예상 효과:**
- 사용 편의성 대폭 향상
- 다양한 시나리오 지원
- 빠른 설정 변경

---

## 4. 🔄 스케줄 재생성 및 버전 비교 기능
**문제점:**
- 생성된 스케줄이 마음에 들지 않아도 수정 불가
- 이전 스케줄과 비교 불가
- 스케줄 생성 히스토리 없음

**개선 방안:**
- "다시 생성" 버튼으로 여러 버전 생성
- 이전 스케줄과 새 스케줄을 나란히 비교
- 스케줄 저장 기능 (생성 날짜별)
- "이 버전으로 저장" 선택 기능
- 스케줄별 만족도 피드백 수집

**예상 효과:**
- 사용자 만족도 향상
- AI 학습 데이터 수집 가능
- 다양한 옵션 중 최적 선택 가능

---

## 5. 🌐 여러 날짜 일괄 계획 및 주간/월간 뷰
**문제점:**
- 한 번에 하루만 계획 가능
- 주간/월간 전망 불가
- 반복 일정 처리 불가

**개선 방안:**
- 날짜 범위 선택으로 여러 날 일괄 계획
- 주간 뷰로 7일간 스케줄 통합 확인
- 월간 뷰로 전체 일정 흐름 파악
- 반복 작업(매일 아침 회의 등) 설정
- 주간 패턴 학습 및 자동 제안

**예상 효과:**
- 장기 계획 수립 가능
- 시간 관리 효율성 향상
- 전체적인 업무 흐름 최적화
